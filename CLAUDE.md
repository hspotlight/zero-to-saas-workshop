# CLAUDE.md — LinkBro

A self-hosted Link-in-Bio web app on Firebase. Single-user, no framework, no bundler.

## Commands

```bash
pnpm install          # Install deps
pnpm serve            # Copy test config + serve on localhost:3000
pnpm test             # Run Jest tests
pnpm test:watch       # Watch mode
pnpm deploy           # Firebase deploy
```

## Architecture

```
linkbro/
├── public/
│   ├── index.html            # Public profile page (/)
│   ├── login.html            # Login page (/login)
│   ├── admin.html            # Admin panel (/admin)
│   ├── style.css             # Structural CSS (design via frontend-design skill)
│   ├── utils.js              # Firebase init + all CRUD helpers + escapeHtml
│   ├── login.js              # Auth: signIn, redirect logic
│   ├── index.js              # Public page: render profile + visible links + analytics
│   ├── admin.js              # Admin: CRUD UI, drag-to-reorder, profile editing, preview
│   ├── firebase-config.js    # Active config (gitignored)
│   └── config/
│       ├── firebase-test.js  # linkbro-test project config
│       └── firebase-prod.js  # linkbro-prod project config
├── __tests__/
│   ├── setup.js              # Global Firebase mock
│   ├── utils.test.js         # escapeHtml, auth, analytics, link validation, reorder
│   └── index.test.js         # Public page rendering, XSS, visibility filtering
└── .github/workflows/
    ├── deploy-test.yml       # PR → linkbro-test
    └── deploy-prod.yml       # Push to main → linkbro-prod
```

## Data Model (Firestore)

```
users/{userId}/profile (single doc "main")
  name: string        (non-empty, max 100)
  bio: string         (max 300, may be empty)

users/{userId}/links/{linkId}
  title: string       (non-empty, max 100)
  url: string         (starts with https://, max 500)
  visible: boolean
  order: number       (integer >= 0, ascending sort)
  createdAt: timestamp (immutable after create)
```

## Module APIs (all in utils.js)

- `signIn(email, password)`, `signOut()`, `onAuthStateChanged(callback)`
- `getProfile(userId)`, `updateProfile(userId, name, bio)`
- `getLinks(userId)`, `addLink(userId, title, url)`, `updateLink(userId, linkId, data)`, `deleteLink(userId, linkId)`
- `setLinkVisibility(userId, linkId, visible)`, `reorderLinks(userId, orderedIds)`
- `trackLinkClick(title, url)`
- `escapeHtml(str)`

## Routing

- `/` — Public profile (no auth, reads profile + visible links)
- `/login` — Login form (redirects to /admin if already authenticated)
- `/admin` — Admin panel (redirects to /login if not authenticated)

## Key Rules

- **XSS**: All user content goes through `escapeHtml()` before DOM insertion
- **Auth guard**: Admin page checks `onAuthStateChanged`, redirects to /login if no user
- **Ordering**: Links sorted ascending by `order`; drag-to-reorder batch-updates all order values
- **Security rules**: Owner-only writes with field-level validation; public reads on profile + links
- **No bundler**: Firebase CDN compat SDK v10.8.0
- **Single user**: One registered account; OWNER_UID set in index.js for public reads

## Testing

- Firebase mocked at boundary in `__tests__/setup.js`
- Tests target public module interfaces and DOM output
- No real Firebase calls in tests

## CI/CD

- PR to main → runs tests → deploys to linkbro-test (OIDC)
- Push to main → runs tests → deploys to linkbro-prod (OIDC)
- Auth via Workload Identity Federation (no stored secrets)

## Config

- `public/config/firebase-test.js` and `firebase-prod.js` are committed (not secrets)
- `public/firebase-config.js` is gitignored, generated by `pnpm serve` or CI
