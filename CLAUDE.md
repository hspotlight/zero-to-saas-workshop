## firebase-webapp-scaffold: What It Produces

This is the primary skill. When invoked, it **interviews the user first** (never jumps straight to code), then scaffolds a full project. The scaffolded project's stack:

- **Frontend**: Plain HTML/CSS/JS — no framework, Firebase SDK loaded from CDN (compat mode v10.8.0); optionally styled with `frontend-design`
- **Auth**: Firebase Authentication (email/password)
- **Database**: Cloud Firestore — all data under `users/{userId}/` by default
- **Hosting**: Firebase Hosting
- **Testing**: Jest + jsdom
- **CI/CD**: GitHub Actions with OIDC (Workload Identity Federation — no stored secrets)
- **Package manager**: pnpm

### Commands in scaffolded projects

```bash
pnpm install          # Install deps
pnpm serve            # Copy test config + serve on localhost:3000
pnpm test             # Run Jest tests
pnpm test:watch       # Watch mode
pnpm deploy           # Firebase deploy
```

### Architecture of scaffolded projects

```
{app-name}/
├── public/
│   ├── utils.js              # Firebase init + all Firestore CRUD helpers
│   ├── login.js              # Auth logic (signIn, register, onAuthStateChanged)
│   ├── {feature}.js          # One JS file per major feature/page
│   ├── style.css
│   ├── firebase-config.js    # Gitignored — generated by copying from config/
│   └── config/
│       ├── firebase-test.js  # Test project config (committed, not secret)
│       └── firebase-prod.js  # Prod project config (committed, not secret)
├── __tests__/
│   ├── setup.js              # Global Firebase mock for Jest
│   └── {feature}.test.js
└── .github/workflows/
    ├── deploy-test.yml       # Deploys to test on PR
    └── deploy-prod.yml       # Deploys to prod on push to main
```

Key principles enforced by the scaffold:
- **User-isolation**: Every Firestore path starts with `users/{userId}/`; security rules enforce this
- **Config strategy**: Environment configs are committed (not secrets); active `firebase-config.js` is gitignored and copied at build/serve time
- **XSS prevention**: `escapeHtml()` used before inserting any user content into the DOM
- **Tests target pure functions**: No Firebase SDK calls in tests — validation, data shapes, and UI helpers only

## tdd Skill Philosophy

- Tests verify behavior through **public interfaces**, not implementation details
- Use **vertical slices** (tracer bullet through the stack), not horizontal layers
- Prefer integration-style tests over unit tests
- Avoid mocking internals — mock only at system boundaries

## Workflow
- TDD: write test first, then implementation
- Vertical slicing: each feature is end-to-end (UI → logic → data → test)
- Keep all Firebase logic in firebase.js
- Never commit .env files
- One feature per branch, PR to merge

## Current Project: Link-in-Bio App

The app being built in this workshop is a **single-user Link-in-Bio web app** (self-hosted Linktree alternative on Firebase). Full PRD is in `PRD-link-in-bio.md`.

### Modules

| Module | File | Responsibility |
|---|---|---|
| Auth | `auth.js` | `signIn(email, password)`, `signOut()`, `onAuthStateChanged(callback)` |
| Links | `links.js` | `getLinks()`, `addLink(title, url)`, `updateLink(id, data)`, `deleteLink(id)`, `setLinkVisibility(id, visible)`, `reorderLinks(orderedIds)` |
| Profile | `profile.js` | `getProfile()`, `updateProfile(name, bio)` |
| Analytics | `analytics.js` | `trackLinkClick(title, url)` — thin wrapper around `logEvent` |
| Public page | `index.js` | Renders visible links + profile; fires `trackLinkClick` on click |
| Admin page | `admin.js` | Full CRUD UI + drag-to-reorder (HTML5 DnD); protected by auth |

### Data Shape (Firestore)

```
users/{userId}/links/{linkId}
  title: string
  url: string
  visible: boolean
  order: number
  createdAt: timestamp

users/{userId}/profile
  name: string
  bio: string
```

### Routing

- `/` — Public profile page (no auth)
- `/admin` — Admin panel (redirects to `/login` if not authenticated)
- `/login` — Login page (redirects to `/admin` if already authenticated)

### Key Constraints

- **XSS**: All user content escaped via `escapeHtml()` before DOM insertion
- **Ordering**: Links sorted ascending by `order`; drag-to-reorder batch-updates all `order` values
- **Security rules**: Writes under `users/{userId}/` require `request.auth.uid == userId`; public reads allowed on profile + links
- **No bundler**: Firebase CDN compat SDK v10.8.0 only
- **Single user**: `userId` is constant at runtime (the one registered account)

### What to Test

- **Links module**: `addLink`, `updateLink`, `deleteLink`, `setLinkVisibility`, `reorderLinks` — verify data shapes and ordering
- **Profile module**: `getProfile`, `updateProfile`
- **Auth module**: `signIn`, `signOut`
- **Analytics module**: `trackLinkClick` calls `logEvent` with correct event name + params
- **Public page**: only visible links rendered, `escapeHtml` applied, click handlers call `trackLinkClick`
